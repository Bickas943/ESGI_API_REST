Documentation de l'API

Introduction

Cette API permet de gérer [précisez ici la fonctionnalité de votre API, par exemple : "la gestion des utilisateurs et des ressources"]. Elle offre une interface complète afin d’effectuer toutes les opérations nécessaires via des requêtes HTTP.

Authentification

Pour accéder aux endpoints de l’API, une authentification est nécessaire. Utilisez votre clé API comme suit :

Authorization: Bearer VOTRE_CLE_API

Liste des Endpoints et Requêtes

1. GET /endpoint1

Description : Récupère la liste des éléments ou une ressource spécifique.

Paramètres

  • param1 (obligatoire) – Description du paramètre.
  • param2 (optionnel) – Description du paramètre.

Exemple de requête

GET /endpoint1?param1=valeur1¶m2=valeur2 HTTP/1.1
Host: api.votredomaine.com
Authorization: Bearer VOTRE_CLE_API

Exemple de réponse

{
  "status": "success",
  "data": {
    /* données retournées */
  }
}

2. POST /endpoint2

Description : Crée une nouvelle ressource.

Paramètres (body)

  • field1 (obligatoire) – Description du champ.
  • field2 (optionnel) – Description du champ.

Exemple de requête

POST /endpoint2 HTTP/1.1
Host: api.votredomaine.com
Authorization: Bearer VOTRE_CLE_API
Content-Type: application/json

{
  "field1": "valeur1",
  "field2": "valeur2"
}

Exemple de réponse

{
  "status": "success",
  "data": {
    /* nouvelle ressource créée */
  }
}

3. PUT /endpoint3/{id}

Description : Met à jour une ressource existante identifiée par son id.

Paramètres

  • id (obligatoire dans l'URL) – Identifiant de la ressource.
  • data (obligatoire dans le body) – Données à mettre à jour.

Exemple de requête

PUT /endpoint3/123 HTTP/1.1
Host: api.votredomaine.com
Authorization: Bearer VOTRE_CLE_API
Content-Type: application/json

{
  "data": "nouvelle valeur"
}

Exemple de réponse

{
  "status": "success",
  "data": {
    /* ressource mise à jour */
  }
}

4. DELETE /endpoint4/{id}

Description : Supprime une ressource identifiée par son id.

Paramètres

  • id (obligatoire dans l'URL) – Identifiant de la ressource à supprimer.

Exemple de requête

DELETE /endpoint4/123 HTTP/1.1
Host: api.votredomaine.com
Authorization: Bearer VOTRE_CLE_API

Exemple de réponse

{
  "status": "success",
  "message": "La ressource a été supprimée"
}

5. PATCH /endpoint5/{id}

Description : Met à jour partiellement une ressource.

Paramètres

  • id (obligatoire dans l'URL) – Identifiant de la ressource.
  • field (obligatoire dans le body) – Champ à mettre à jour.

Exemple de requête

PATCH /endpoint5/123 HTTP/1.1
Host: api.votredomaine.com
Authorization: Bearer VOTRE_CLE_API
Content-Type: application/json

{
  "field": "valeur mise à jour"
}

Exemple de réponse

{
  "status": "success",
  "data": {
    /* ressource partiellement mise à jour */
  }
}

6. OPTIONS /endpointX

Description : Permet de connaître les méthodes HTTP autorisées pour l’endpoint.

Exemple de requête

OPTIONS /endpointX HTTP/1.1
Host: api.votredomaine.com
Authorization: Bearer VOTRE_CLE_API

Exemple de réponse

HTTP/1.1 200 OK
Allow: GET, POST, PUT, DELETE, PATCH, OPTIONS

Gestion des erreurs

En cas d’erreur, l’API renvoie un code HTTP approprié ainsi qu’un objet JSON contenant un message descriptif. Par exemple :

{
  "status": "error",
  "message": "Description de l'erreur"
}
