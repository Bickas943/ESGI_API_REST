Documentation API – Utilisation via Postman (URIs uniquement)

Ce document présente l'ensemble des URI de l'API à utiliser dans Postman. Il n'est question ici que de l'accès aux endpoints via les URI (sans aborder l'interface web) et de la gestion des variables d'environnement. Vous trouverez pour chaque endpoint les variables requises, les paramètres acceptés et des exemples concrets d'utilisation.

1. Configuration de Base

Définissez dans Postman les variables d'environnement suivantes :

  • {{base_url}} : URL de base de l'API (ex. http://votre-domaine.com/api)
  • {{auth_token}} : Token d'authentification (si nécessaire)

2. Authentification

POST {{base_url}}/auth/login

Permet de se connecter et d'obtenir un token d'authentification pour accéder aux autres endpoints.

Requête (body, format JSON) :

{
  "username": "votre_utilisateur",
  "password": "votre_mot_de_passe"
}

Réponse (exemple) :

{
  "token": "votre_token_jwt",
  "expires_in": 3600
}

3. Endpoints Principaux

GET {{base_url}}/items

Récupère la liste de tous les items disponibles.

Paramètres optionnels :

  • ?page= : numéro de la page (ex : ?page=1)
  • ?limit= : nombre d'items par page (ex : ?limit=10)

Exemple d'appel :

GET {{base_url}}/items?page=1&limit=10

GET {{base_url}}/items/{id}

Récupère un item spécifique grâce à son identifiant.

Paramètre URI :

  • {id} : Identifiant de l'item

Exemple d'appel :

GET {{base_url}}/items/123

POST {{base_url}}/items

Crée un nouvel item dans la base de données.

Corps de la requête (format JSON) :

{
  "name": "Nom de l'item",
  "description": "Description détaillée",
  "price": 19.99
}

Exemple d'appel :

POST {{base_url}}/items

PUT {{base_url}}/items/{id}

Met à jour les informations d'un item existant.

Paramètre URI :

  • {id} : Identifiant de l'item à mettre à jour

Corps de la requête (format JSON) :

{
  "name": "Nouveau nom",
  "description": "Nouvelle description",
  "price": 24.99
}

Exemple d'appel :

PUT {{base_url}}/items/123

DELETE {{base_url}}/items/{id}

Supprime un item de la base de données.

Paramètre URI :

  • {id} : Identifiant de l'item à supprimer

Exemple d'appel :

DELETE {{base_url}}/items/123

4. Endpoints Complémentaires

GET {{base_url}}/categories

Récupère la liste des catégories disponibles.

Exemple d'appel :

GET {{base_url}}/categories

POST {{base_url}}/categories

Crée une nouvelle catégorie.

Corps de la requête (format JSON) :

{
  "name": "Nom de la catégorie"
}

Exemple d'appel :

POST {{base_url}}/ca
